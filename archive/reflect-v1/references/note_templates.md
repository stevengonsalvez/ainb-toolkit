# Note Templates

Templates for learning notes, episode notes, and entity sidecars generated by /reflect.

## Learning Note Template

```yaml
---
type: learning
id: lrn-{slug}-{hash6}
created: {ISO timestamp}
updated: {ISO timestamp}
scope: universal | domain:{x} | project:{x}
confidence: high | medium | low
learning_type: pattern | correction | bug-fix | decision | anti-pattern
title: "{title}"
tags: [tag1, tag2]
symptoms:
  - "{symptom1}"
  - "{symptom2}"
key_insight: "{THE ONE THING}"
links: []
source_episodes: [ep-{date}-{hash}]
superseded_by: null
---

## Problem
{description of the problem encountered}

## Solution
{step-by-step solution with code examples}

## Anti-Pattern
{what to avoid, if applicable}

## Context
{additional context, why this happened, how to prevent}
```

## Episode Note Template

Episode notes are raw session snapshots that provide provenance.
They are auto-created and do NOT require user approval.

```yaml
---
type: episode
id: ep-{YYYYMMDD}-{hash6}
created: {ISO timestamp}
agent: {agent-name}
project: {detected-project}
tags: [relevant-tags]
extracted_learnings: [lrn-ids]
---

## Session Context
{what happened in the session - brief narrative}

## Signals Detected
{list of signals found, with confidence}

## Learnings Extracted
{list of learning IDs generated from this episode}
```

## Behavioral Change Proposal Format

```markdown
### Change N: Update [agent-name]

**Target**: `[file path]`
**Section**: [section name]
**Confidence**: [HIGH/MEDIUM/LOW]
**Rationale**: [why this change]

```diff
--- a/path/to/agent.md
+++ b/path/to/agent.md
@@ -82,6 +82,7 @@
 ## Section

 * Existing rule
+* New rule from learning
```
```

## Project-Scoped Output

Project-scoped learnings are also saved to `docs/solutions/`:

```bash
mkdir -p docs/solutions/{category}
# Save: docs/solutions/{category}/{descriptive-name}.md
# Sidecar: docs/solutions/{category}/{descriptive-name}.entities.yaml
```
