# Composition patterns, format presets, and originality rules

## Format presets

Pick by destination; state the aspect explicitly in every prompt.

| Use case | Aspect | Adjustments |
|----------|--------|-------------|
| Article/blog body | 16:9 | default everything |
| Social post (LinkedIn/X) | 1:1 or 4:5 | bigger labels, ONE punchline line allowed, single bold idea |
| README/website hero | 21:9 or 16:9 | extra quiet space on one side for overlay text |
| Slides | 16:9 | keep top ~15% calm for slide titles |
| Architecture explainer | 16:9 series | see architecture-series pattern |

## Base structure types

Pick one structure per image; don't mix.

### Workflow process
Input -> process -> output. Popa or a wonky machine in the middle; green arrows for the main flow.

### System fragment
3-5 core modules only; Popa performing one key action inside one of them.

### Before/after contrast
Chaos left, calm right, green arrow between. Popa appears in both halves, doing the same job badly then well.

### Role state
2-4 small Popa states, one short label each (stuck / caffeinated / shipped).

### Concept metaphor
One big weird low-tech object or machine, few inputs, one output. Memorable over complete.

### Method layering
Stacked boxes, never a formal pyramid; Popa hauling or inspecting a layer.

### Map route
One winding path, few nodes, Popa walking it with the notepad.

### Mini comic panels
2-4 small scenes, one action per panel. Good for failure-to-success arcs.

## Architecture-series pattern (strict minimal, split big topics)

Never one dense diagram. A big system becomes a SERIES:

1. **Overview frame**: 3-5 mega-blocks max, drawn as friendly low-tech objects (buildings, crates, machines). Popa as tour guide pointing with the pen.
2. **Zoom-in frames**: one per block that matters, again 3-5 elements each, Popa operating the relevant part.
3. Optional **flow frame**: the one critical path through the system as a map route.

Each frame stands alone; together they read as chapters. Number them in the shot list.

## How to generate an original metaphor

Reinvent per image from the current source; never copy old images.

### Three steps

1. Swap the abstract concept for a physical action: stuck, dropped, weighed, sorted, fermented, sealed, leaked, rerouted, watered, archived.
2. Swap the system for a low-tech object: crate, drawer, greenhouse, mail chute, dumbwaiter, scale, well, conveyor, filing cabinet, sluice gate, vending machine.
3. Let Popa carry the action — inside the machine, at the gate, on the ladder — notepad present, logging it.

### Object pool

Crate, drawer, greenhouse, watering can, mail chute, scale, well, ladder, pipe, string-and-pins board, gate, turntable, stamp, hole punch, clothesline, tiny desk with too many trays. Pick 1-2 per image.

### Popa action pool

Crank, weigh, file, stamp, water, sort, seal, patch, guard, lower, catch, thread, inspect, log, tour-guide. The action serves the core idea — never weird for weird's sake.

## Anti-copy rules

`assets/examples/` is for style calibration only (line density, pastel tone, label restraint, Popa's temperament). Unless the user explicitly says "reproduce this one," do not reuse example compositions. Same-topic requests get new metaphors: a "deploy pipeline" doesn't have to be a conveyor — it can be Popa raising seedlings in a greenhouse and shipping the pots.
