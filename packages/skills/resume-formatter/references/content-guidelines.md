## Profile Section

### Structure
- 3-4 sentences maximum
- First sentence: Role identity + primary domain
- Second sentence: Key expertise areas
- Third sentence: Scale/impact indicators
- Optional fourth: Differentiator or unique value

### Example
```
Engineering leader specialising in customer experience platforms that drive discovery,
engagement, and conversion at e-commerce scale. Deep expertise building homepage,
product listing, and product detail page experiences across web and app—spanning
personalisation engines, search & navigation, and experimentation frameworks.
Proven track record leading multi-squad engineering teams to deliver immersive,
high-performance digital experiences that delight millions of customers.
```

### Tailoring Tips
- Mirror key terminology from job description
- Match industry language (e-commerce, fintech, etc.)
- Include specific domain expertise relevant to role

## Experience Section

### Achievement Bullet Pattern

**Format**: `<strong>Headline</strong>: Detailed description with metrics.`

**Good Examples**:
- `<strong>Platform transformation</strong>: Architected unified platform serving 3 brands with shared components; supports £9.8bn revenue division.`
- `<strong>Search & navigation</strong>: Improved relevancy/CTR by ~11%, reduced p95 latency by 27% through index optimization and CDN tuning.`

**Avoid**:
- Starting with "Responsible for..."
- Vague statements without metrics
- Technical jargon without context

### Metrics to Include

| Category | Examples |
|----------|----------|
| Scale | Team size, budget, revenue, customers, MAU |
| Performance | Latency improvements, uptime, error rates |
| Efficiency | Cost savings, time reductions, automation gains |
| Quality | DORA metrics, crash-free rate, app ratings |
| Growth | Conversion uplift, engagement increase, adoption rates |

### Scale Snapshot Box

Use for significant roles to highlight key numbers at a glance:

```
Scale snapshot: £9.8bn revenue • 25m customers • 170 engineers •
99.98% availability • 4.6★ App Store
```

## Skills Section

### Category Organization

Organize skills by domain, not by type:

**Good Categories**:
- Customer Experience Engineering
- Engineering Leadership
- Platforms & Developer Experience
- Frontend & App
- Backend & Architecture
- Infrastructure

**Avoid**:
- "Hard Skills" / "Soft Skills"
- Overly generic categories
- Too many categories (max 6-7)

### Skills Format

- Use industry-standard terminology
- Include acronyms where appropriate (CI/CD, SLIs/SLOs)
- Group related items together
- 8-15 items per category

## Education Section

### Format
```html
<div class="education-entry">
    <div class="degree">Bachelor of Engineering</div>
    <div class="university">University Name</div>
    <div class="years">2002-2006</div>
</div>
```

### What to Include
- Degree name and field
- Institution name
- Years attended
- Relevant certifications (optional, separate entries)

### What to Omit
- GPA (unless exceptional and recent graduate)
- Coursework details
- High school education

## ATS Optimization

### Keyword Matching
- Use exact terminology from job description
- Include both spelled-out terms and acronyms
- Mirror the job title language

### Common Synonyms to Include

| JD Term | Also Include |
|---------|--------------|
| CI/CD | Continuous Integration, Continuous Deployment |
| Kubernetes | K8s |
| AWS | Amazon Web Services |
| Agile | Scrum, Kanban |
| GraphQL | GQL, federated GraphQL |

### Formatting for ATS
- Use standard section headers (Profile, Experience, Skills, Education)
- Avoid tables for critical content
- Keep formatting simple and consistent
- Use standard fonts (Inter, Arial, Helvetica)

### Length Guidelines
- Target: 2-3 pages for senior roles
- Entry-level: 1 page
- Executive: 2-3 pages maximum
- Never exceed 3 pages
